为Kubernetes Federation API生成参考文档

本页面将展示如何自动为Kubernetes Federation API生成相关参考文档。

  • Before you begin
  • 运行update-federation-api-docs.sh脚本
  • What’s next

Before you begin

  • 你需要安装好Git

  • 你需要安装1.9.1 或者更高版本的Golang,同时你的环境变量中需要将$GOPATH设置正确

  • 你需要知道如何去创建一个PR.例如,你首先需要将远程仓库fork到你的个人仓库.详见创建一个文档相关的PR

运行update-federation-api-docs.sh脚本

如果你本地还没有Kubernetes federation源码,请先下载：

mkdir $GOPATH/src
cd $GOPATH/src
go get github.com/kubernetes/federation

确定你kubernetes/federation项目的根目录.例如,如果你执行了上面的命令，你的根目录应该为$GOPATH/src/github.com/kubernetes/federation下文将以<fed-base>指代项目根目录.

运行文档自动生成脚本:

cd <fed-base>
hack/update-federation-api-reference-docs.sh

脚本将运行 gcr.io/google_containers/gen-swagger-docs 镜像生成以下相关文档:

  • /docs/api-reference/extensions/v1beta1/operations.html
  • /docs/api-reference/extensions/v1beta1/definitions.html
  • /docs/api-reference/v1/operations.html
  • /docs/api-reference/v1/definitions.html

自动生成的文档不会自动发布,你需要手动将他们拷贝到 kubernetes/website 项目.

以下文档发布在 kubernetes.io/docs/reference:

  • Federation API v1 操作
  • Federation API v1 定义
  • Federation API extensions/v1beta1 操作
  • Federation API extensions/v1beta1 定义
