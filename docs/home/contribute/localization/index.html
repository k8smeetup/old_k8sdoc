本地化 Kubernetes 文档

我们很高兴将 Kubernetes 文档的本地化（l10n）添加到 website！

本地化必须满足以下 工作流程 要求（如何本地化）和 输出（本地化的内容）。

工作流程

所有 l10n 工作必须在 Kubernetes 组织 内进行存储和跟踪.

本地化的基础

本地化必须从 最近发布的主要版本 的英文文件中获得。

注意： 想要查找最新发布版本的文档的源文件：

  1. 导航到 https://github.com/kubernetes/website。

  2. 选择 release-1.X 最新版本的分支。

    例如，Kubernetes v1.9 文档的分支是 release-1.9。

源文件驻留在 /docs/ 目录中。

仓库

一个 l10n 团队将有一个专门用于其工作的仓库，例如：kubernetes/kubernetes-docs-cn。

注： 想要打开一个 l10n 仓库，请在 Slack 上 联系 SIG docs 的负责人 寻求帮助。

项目

团队必须使用 GitHub 项目 来跟踪他们的整体进度。

项目必须包含以下列：

  • 需要做
  • 进行中
  • 已完成

例如：中文本地化项目。

团队功能

l10n 团队必须提供单点联系人：可以回应或重定向问题或疑虑的人员的姓名和联系信息。

l10n 团队必须提供自己的仓库维护人员。

所有 l10n 的工作必须能够与团队自身的资源保持一致

只要有可能，每个本地化页面都必须由来自不同公司的翻译人员批准。

上游捐款

上游捐款受到欢迎和鼓励！

为了提高效率，将上游贡献限制为每周一个 pull 请求，其中包含一个压缩的提交。

输出

所有本地化必须至少包含以下文档：

描述 URL
Home 所有标题和次标题的 URL
Setup 所有标题和次标题的 URL
Tutorials Kubernetes 基础，Hello Minikube

下一步

一旦 l10n 满足工作流程和最低输出要求，SIG docs 将会：

  • 与本地化团队合作，在网站上实施语言选择。
  • 通过 云计算基金会（CNCF）渠道宣传可用性。

注意： 语言选择的实施正在等待 Kubernetes 完成的第一个本地化项目。
